# **TEProF2_2026**

(**T**ransposable **E**lement **Pro**moter **F**inder **2**, 2026 release)

TEProF2_2026 is a Python 3.12 rewrite of the [TEProF2](https://doi.org/10.1038/s41588-023-01349-3) pipeline, adapted for non-model plant transcriptomes with fragmented reference genomes and large proportions of unclassified TE families. It identifies TE-promoter-driven chimeric transcripts from RNA-seq data assembled by StringTie/TACO, with two complementary detection modes and integrated DeepTE-based TE reclassification.

This version was developed for the analysis of *Ambrosia artemisiifolia* (common ragweed) transcriptomes across 22 population groups (15 North American native + 7 Chinese invasive).

Please cite the original TEProF2: https://doi.org/10.1038/s41588-023-01349-3

# Summary

TEProF2_2026 v3.0.0

This pipeline takes population-level assembled RNA-seq data (TACO-merged `.gtf` format) and identifies transcripts whose transcription start sites (TSS) are driven by transposable element promoters. It outputs standardized 49-column TSV files with per-transcript TE annotation, supporting both strict (TSS inside TE) and window (TE upstream of TSS) detection modes.

# Key Changes from Original TEProF2

| Feature | Original TEProF2 | TEProF2_2026 |
|---------|-----------------|--------------|
| Language | Python 2.7 | Python 3.12 |
| Input unit | Per-sample StringTie GTF | Per-group TACO-merged GTF |
| TE classification | RepeatMasker subfamily/family/class | RepeatMasker + DeepTE superfamily reclassification |
| Detection modes | Single window | Strict (TSS ∩ TE ≤ 1 bp) + Window (TE within 500 bp upstream) |
| Simple repeat handling | Included in hits | Filtered out (not in `repeat_classification.tsv`) |
| Output format | Variable columns, no header | Fixed 49-column TSV with `te_driven_mode` field |
| CLI | Script with `arguments.txt` | `teprof3` CLI with named arguments |


## (2) Reference Files

### (A) RepeatMasker BED (bgzipped + tabix indexed)

Run RepeatMasker on your genome, then:

```bash
# Sort, bgzip, and index
sort -k1,1 -k2,2n rmsk.bed > rmsk_sorted.bed
bgzip rmsk_sorted.bed -o rmsk.bed.gz
tabix -p bed rmsk.bed.gz
```

Both `rmsk.bed.gz` and `rmsk.bed.gz.tbi` must be in the same directory.

### (B) TE Classification Table

A tab-delimited file mapping TE family name to class and family. Generated by RepeatModeler or DeepTE reclassification:

```
# format: name  class  family
rnd-1_family-1  LTR  Gypsy
rnd-1_family-2  DNA  hAT-MITE
...
```

Simple repeats (e.g. `(CT)n`) should **not** be included — the pipeline uses this file to filter non-TE records from the rmsk BED.

### (C) Gencode Dictionaries

Two pickle/dictionary files for plus and minus strand gene annotation, generated from a reference GTF:

```bash
# From original TEProF2 tools:
cat reference.gtf | awk '{if($3=="transcript"||$3=="exon"||$3=="start_codon"){print}}' \
    | awk -F "; " '{print $0"\t"$2}' > reference_sorted.gtf
genecode_to_dic.py reference_sorted.gtf
# Produces: genecode_plus.dic  genecode_minus.dic
```

## (3) Input Files

Per-group merged GTF files produced by TACO from StringTie per-sample assemblies:

```bash
# Step 1: StringTie per-sample assembly
stringtie sample.bam -p 50 -G reference.gtf -o sample.gtf

# Step 2: TACO per-group merge
taco_run -p 40 group_gtf_list.txt -o GROUP_merged/
# Input to TEProF2_2026: GROUP_merged/assembly.gtf
```

# Usage

## (1) Installation

```bash
git clone https://github.com/lilim9527/TEProF2_2026.git
cd TEProF2_2026
pip install -e .
```

Verify:

```bash
teprof3 --help
```

## (2) Prepare Reference Files

Ensure the following files are ready:

```
rmsk.bed.gz                  # bgzipped RepeatMasker BED
rmsk.bed.gz.tbi              # tabix index (same directory)
repeat_classification.tsv    # TE name -> class/family mapping
genecode_plus.dic            # plus-strand gene annotation dict
genecode_minus.dic           # minus-strand gene annotation dict
```

## (3) Run Annotation

**Single group:**

```bash
teprof3 GROUP_merged/assembly.gtf \
    rmsk.bed.gz \
    repeat_classification.tsv \
    genecode_plus.dic \
    genecode_minus.dic \
    --strict-bp 1 \
    --window-bp 500 \
    --sample GROUP_NAME \
    --output-dir results/
```

**Batch (all groups, 4 parallel workers):**

```python
from concurrent.futures import ProcessPoolExecutor, as_completed
import subprocess, sys

GROUPS_CN = ['DD', 'FK', 'HG', 'MDJ', 'NC', 'WH', 'WP']
GROUPS_NA = [str(i) for i in range(1, 16)]
GTF_CN = "OPE_DATA/Input_GTFs/CN"
GTF_NA = "OPE_DATA/Input_GTFs/NA"
BASE_TE = "operator/LIB_Build/runs/ragweed_with_deepte"
GENCODE_PLUS = "LIB_Build/genecode_plus.dic"
GENCODE_MINUS = "LIB_Build/genecode_minus.dic"
OUT = "results/TEProF2_v3/"

jobs = [(g, f"{GTF_CN}/{g}.gtf") for g in GROUPS_CN] + \
       [(g, f"{GTF_NA}/{g}.gtf") for g in GROUPS_NA]

def run_group(group, gtf):
    cmd = ["teprof3", gtf,
           f"{BASE_TE}/rmsk.bed.gz",
           f"{BASE_TE}/repeat_classification.tsv",
           GENCODE_PLUS, GENCODE_MINUS,
           "--strict-bp", "1", "--window-bp", "500",
           "--sample", group, "--output-dir", OUT]
    r = subprocess.run(cmd, capture_output=True, text=True)
    return group, r.returncode

with ProcessPoolExecutor(max_workers=4) as ex:
    for group, rc in [f.result() for f in as_completed(
            {ex.submit(run_group, g, gtf): g for g, gtf in jobs})]:
        print(f"{'OK' if rc == 0 else 'FAIL'}: {group}")
```

## (4) Output Files

Three TSV files are produced per group in `--output-dir`:

```
results/
├── GROUP_all.tsv       # all transcripts (49 columns)
├── GROUP_strict.tsv    # strict-mode hits only
└── GROUP_window.tsv    # window-mode hits only
```

# Detection Modes

| Mode | Criterion | Biological interpretation |
|------|-----------|--------------------------|
| `strict` | TSS overlaps TE body (≤ 1 bp gap) | TE provides internal Pol II promoter (LTR/LINE) |
| `window` | TE within 500 bp upstream of TSS | TE acts as upstream regulatory element (MITE/DNA transposon) |
| `none` | No TE hit in either mode | No TE-driven evidence |

The `te_driven_mode` column in the output encodes the final call:
- `strict` — strict hit AND TE class is LTR or LINE
- `window` — window hit (or strict hit with non-LTR/LINE class)
- `none` — no hit

# Output Column Reference

Key columns in the 49-column TSV:

| Column | Description |
|--------|-------------|
| `transcript_id` | Transcript identifier from input GTF |
| `chrom`, `start`, `end`, `strand` | Genomic coordinates |
| `n_exons`, `splicing` | Exon count and splicing status (Yes/No) |
| `tss_element` | Genomic element at TSS (exon/intron/intergenic) |
| `strict_hit` | Boolean: TSS directly overlaps TE |
| `strict_te_name` | TE family ID (strict mode) |
| `strict_te_class` | TE class after DeepTE reclassification |
| `strict_te_family` | TE superfamily |
| `strict_n_te_overlaps` | Number of TE records overlapping TSS |
| `window_hit` | Boolean: TE within 500 bp upstream |
| `window_te_name` | TE family ID (window mode) |
| `window_te_class` | TE class (window mode) |
| `window_te_family` | TE superfamily (window mode) |
| `window_n_te_overlaps` | Number of TE records in window |
| `te_driven_mode` | Final call: `strict` / `window` / `none` |
